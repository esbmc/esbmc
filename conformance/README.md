## What this package contains

This directory (`conformance/`) contains all scripts and data to reproduce
the P4a translation soundness experiments reported in Table~\ref{tab:conformance}.

```
conformance/
├── README.md                  ← this file
├── gen_inputs.py              ← Step 1: generate concrete input sequences
├── plc_simulator.py           ← Step 2: IEC 61131-3 Python reference simulator
├── concrete_executor.py       ← Step 3: GOTO-IR concrete executor (source of truth)
├── diff_outputs.py            ← Step 4: cycle-by-cycle diff and LaTeX table
├── run_all.sh                 ← One-command reproduction script
├── inputs/                    ← Pre-generated input sequences (reproducible, seed=42)
│   ├── manifest.json
│   ├── motor_interlock_inputs.json
│   ├── traffic_light_safe_inputs.json
│   ├── bottle_filling_safe_inputs.json
│   ├── elevator_safe_inputs.json
│   ├── water_control_inputs.json
│   ├── stairs_light_inputs.json
│   └── tank_level_safe_inputs.json
├── esbmc_goto_outputs/        ← GOTO-IR executor outputs (generated by run_all.sh)
└── expected/                  ← Expected results for CI verification
    └── conformance_report.json
```

## Step-by-step reproduction

### Step 1 — Generate input sequences

```bash
cd ~/esbmc
python3 conformance/gen_inputs.py --all --output conformance/inputs/
```

This generates 7 JSON files, one per benchmark, each with:
- 50 random input sequences
- 10 scan cycles per sequence
- Reproducible via `--seed 42` (default)
- Physical constraints respected (e.g., HIGH_SWITCH and LOW_SWITCH never both 1)

### Step 2 — Run the GOTO-IR concrete executor

```bash
python3 conformance/concrete_executor.py \
    --inputs conformance/inputs/ \
    --output conformance/esbmc_goto_outputs/
```

This extracts the `scan_loop` from each benchmark's GOTO IR using
`--goto-functions-only`, then re-executes it with concrete input values.

**Important implementation notes:**
- Multiple `ASSIGN` to the same variable = parallel rungs = OR semantics
- The coil-energise prefix `1 &&` is stripped only at the **start** of each
  expression (not globally) to avoid incorrect evaluation
- Persistent state (timer counters, latch coils) is maintained across scans

### Step 3 — Check safety properties and generate report

```bash
python3 conformance/diff_outputs.py \
    --esbmc   conformance/esbmc_goto_outputs/ \
    --inputs  conformance/inputs/ \
    --report  conformance/conformance_report.json
```

Outputs:
- `conformance/conformance_report.json` — machine-readable results
- LaTeX table rows ready to paste into Table~\ref{tab:conformance}

## Known limitations

### Graphical PLCopen XML (tc6_0201)

Two benchmarks use the graphical LD format produced by OpenPLC Editor v3:

| Benchmark | Source | Format |
|---|---|---|
| `water_control` | CONTROLLINO-PLC/OpenPLC_examples (MIT, 2024) | tc6_0201 graphical |
| `stairs_light` | CONTROLLINO-PLC/OpenPLC_examples (MIT, 2024) | tc6_0201 graphical |

The SAFE-LD parser accepts these files (VERIFICATION SUCCESSFUL), but the
LD→GOTO-IR converter does not yet emit rung logic for coordinate-based
connections (`localId`/`refLocalId`). Completing this conversion is planned
as immediate future work.

### Single-scan model

The concrete executor models each scan independently with fresh non-deterministic
inputs. This matches the ESBMC scan model for `invariant`, `mutual_exclusion`,
and `absence` properties. Properties of kind `response` (bounded liveness)
require multi-scan traces and are not covered by this conformance test.

## Reproducing on a different machine

The experiments were run on:
- Apple Silicon M-series (aarch64), macOS Tahoe 15.4
- ESBMC v8.3.0, Z3 v4.16.0

To reproduce on Linux (x86_64):

```bash
# Build ESBMC for Linux following BUILDING.md
# Then update the ESBMC path if needed:
python3 conformance/concrete_executor.py \
    --esbmc ./build/src/esbmc/esbmc \
    --inputs conformance/inputs/ \
    --output conformance/esbmc_goto_outputs/
```

The Python scripts have no platform-specific dependencies.

## Docker (optional)

For fully reproducible results in a clean environment:

```bash
# Build the ESBMC Docker image (from repo root)
docker build -t esbmc:8.3.0 .

# Run the conformance tests inside Docker
docker run --rm \
    -v $(pwd):/esbmc \
    -w /esbmc \
    esbmc:8.3.0 \
    bash conformance/run_all.sh
```
